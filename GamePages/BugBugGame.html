BugBug - a Po-op Game

About Project

This is a game I made with 11 other people where the focus was on making a fun game and improving our skills. This is a chaotic 2-player co-op game (can be played solo as well) where the players need to communicate and work together to steer a dungball through different levels. There are 7 levels, each getting more difficult with new enemy encounters and more difficult terrain. Some levels even add more features to the dungball such as a shield.

Highlights

  • Local co-op support
  • FMOD sound/music support
  • Effect system
  • Physics based movement
  • Screen indicators
  • UI and options systems

Local co-op support

I'm really proud of the implementation that handles multiple players by using Unity’s new input system. When a player joins a new input object is created which is connected to a controller. This object only handles input and isn’t destroyed when switching scenes. When a scene is loaded the actual player objects for that scene are connected to a specific input object. In the player menu the player can change the color of their bug and the same strategy is used here, by connecting the menu to the input object.

Effect system

The effect system has a high level of abstraction by having all effects inherit from a base class which is the only thing the effect system knows about. The only thing the effect system does is activate and deactivate effects without caring about what the effect is. There are 5 different effects that affect the dungball in different ways, for example taking tick damage or making the ball slide more on ice.

FMOD

This was my first time using a different sound engine than the one Unity is provided with. I learned a lot on how to implement FMOD for both sound effects and music, I also learned a little about how FMOD Studio works.
