Marching Cube Terrain Generation

About Project

Procedural terrain generator that I made using the marching cube algorithm and perlin noise. The goal of this project was to learn more about terrain generation, destructible terrain and how to optimize them. In this project an infinite world is generated around the player using a chunk system. The terrain can be modified by breaking or placing blocks.

Highlights

  • Marching Cubes
  • Procedural Generation
  • Perlin Noise
  • Chunk system
  • Unity Jobs System
  • Unity Burst Compiler
  • Object Pooling

Procedural Generation

The generation is infinite in all axes by using a chunk system to split the terrain into different meshes. I used perlin noise with multiple octaves for the terrain data and by using the marching cube algorithm the terrain could easily be made modifiable letting the player both place and destroy blocks. The marching cube algorithm has no smoothing leading to the sharp edges which i think looks really cool (:

Optimizations

To optimize the generation, multithreading was used through Unity Jobs and to further improve performance the burst compiler was used. This made a huge difference to performance going from ~140ms to load in 64 chunks to ~6ms. The second performance problem was garbage collection, to fix this I used object pooling for the chunks. That made most of the problem disappear leaving the generation with no big stutters when loading in terrain. Next step is to implement octrees for chunks so that the marching cube algorithm doesn't waste its time going through completely empty/filled chunks.
